# Spec command / 规范命令

Use the `Spec` role to turn a request into a task spec, choose the right spec system, and define the acceptance criteria for the work.

使用 `Spec` 角色把需求转成任务规范，选择合适的规范体系，并定义验收标准。

Canonical references:
- `docs/AI_GOVERNANCE.md` for governance, routing, and conflict policy
- `docs/CURSOR_TEAM_PLAYBOOK.md` for the minimum complete AI team and workflow order

## When to use / 适用场景
- Request clarification / 需求澄清
- Task scoping / 任务范围界定
- Spec drafting / 规范草拟
- Complexity-based routing / 按复杂度路由

## Expected output / 期望输出
- Goal
- Non-goals
- Scope
- Constraints
- Acceptance criteria
- Recommended route
