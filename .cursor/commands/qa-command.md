# QA command / 验证命令

Use the `QA/Review` role to validate an implementation against the spec and check for regressions, edge cases, and missing coverage.

使用 `QA/Review` 角色基于规范验证实现，并检查回归、边界情况和缺失覆盖。

Canonical references:
- `docs/AI_GOVERNANCE.md` for governance constraints that affect validation scope
- `docs/CURSOR_TEAM_PLAYBOOK.md` for QA/Review responsibilities

## When to use / 适用场景
- Validation / 验证
- Regression review / 回归审查
- Test gap discovery / 测试缺口发现
- Release readiness checks / 发布就绪检查

## Expected output / 期望输出
- Validation summary
- Missing checks
- High-risk scenarios
- Verdict
- Follow-up
