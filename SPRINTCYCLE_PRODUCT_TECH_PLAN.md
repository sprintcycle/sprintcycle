## 一、产品定位与市场机会

### 1.1 核心定位

面向所有用户的 AI 驱动全栈开发平台，通过智能化敏捷多轮实践实现生产级高质量代码和服务。

对标 Y Build 但走开源可自部署路线，填补企业级可控 AI 开发工具的市场空白。

|维度|定位描述|
|---|---|
|一句话|具备自进化能力的敏捷开发框架，采用「意图驱动」模式，通过多轮Sprint闭环实现自动化迭代|
|目标用户|全用户覆盖 — 从非技术创始人、产品经理，到专业开发者、企业技术团队|
|核心差异|开源 + 可自部署 + 代码质量优先 + 100% 数据可控|
|价值主张|从 "快速上线能用" 到 "持续进化的生产级代码"|

### 1.2 市场机会分析

|竞品|优势|劣势|SprintCycle 机会|
|---|---|---|---|
|Y Build|上手快、一键部署、增长优化|闭源黑盒、代码质量不可控、数据在第三方、无自部署|✅ 开源可自部署、代码质量体系、企业级安全|
|Cursor / Windsurf|IDE 集成、编辑体验好|缺乏端到端项目管理、无架构保护、难以团队协作|✅ 全流程闭环、架构守护、团队协作能力|
|Aider / Claude Code|命令行友好、Git 集成|缺乏项目级规划、无测试闭环、单人工具为主|✅ 项目级管理、完整测试体系、可扩展平台|
|Devin / Cognition|全流程自动化|超级昂贵（$1000+/月）、闭源、等待名单|✅ 开源免费、可自部署、透明可控|

**核心机会**：市场上缺乏一个 开源 + 可自部署 + 企业级 + 代码质量优先 的全栈AI开发平台。

### 1.3 关键区分 ✨（重要修正）

|场景|策略|质量等级|适用对象|
|---|---|---|---|
|SprintCycle 自身演进|质量守护体系优先|四级质量保证强制生效|SprintCycle 框架开发|
|客户产品开发 (product)|快速上线优先，兼顾质量守护|可配置质量等级（默认为 Level 2）|用户业务系统、应用开发|

**设计哲学**：
- 框架本身必须绝对可靠 → 严格质量闭环
- 用户产品需要快速验证 → 灵活质量档位，根据成熟度逐步提升

## 二、核心能力体系

### 2.1 三大核心能力支柱

#### 🔑 支柱一：意图驱动开发（Intention-Driven）

- 自然语言需求输入：用户用自然语言描述产品需求
- 自动需求拆解：系统自动拆分为可执行的开发任务
- 上下文感知：理解项目历史、代码结构、业务逻辑
- 多轮迭代优化：支持持续对话式改进

**能力要点**：
- ✅ 非技术用户友好：自然语言描述需求即可
- ✅ PRD 自动生成：从意图自动转化为结构化需求文档
- ✅ 多轮确认机制：关键决策点人工介入确认
- ✅ 上下文持久化：历史意图和决策可追溯

#### 🔑 支柱二：多编码引擎支持

采用策略模式实现多引擎兼容，灵活切换，不绑定某一个模型。

|引擎|特点|适用场景|
|---|---|---|
|Aider|终端驱动，代码修改精准|核心功能开发|
|Claude Code|长上下文理解，架构设计|系统设计、重构|
|Cursor SDK|IDE 原生体验，实时协作|日常编码辅助|
|OpenLLM|开源可自部署，数据可控|企业隐私场景|

#### 🔑 支柱三：智能质量保证（Quality Guardian）

**质量等级可配置（客户产品专属）**：
- **Level 0**: 仅生成代码，无质量检查（极速原型）
- **Level 1**: 静态检查 + 语法验证（默认）
- **Level 2**: 单元测试 + 基本架构验证（推荐）
- **Level 3**: 四级质量守护全量生效（生产级）

**四级质量守护（SprintCycle 自身强制全量生效）**：

|层级|检查内容|执行时机|
|---|---|---|
|L1 架构不变量守护|ArchitectureGuard 验证核心架构约束|每次提交前|
|L2 适应度评估|EvolutionFitness 评估代码质量改进|变更后自动|
|L3 测试闭环|自动生成测试用例，覆盖率达标|功能完成后|
|L4 静态代码检查|Ruff + Mypy 严格规范|实时执行|

### 2.2 双模式执行体系

|模式|目标|风险等级|人工介入点|适用场景|
|---|---|---|---|---|
|Normal Mode<br/>正常开发模式|按用户意图开发新功能、修复Bug|🟡 中等|Sprint 规划确认、结果验收|日常功能开发、迭代|
|Evolution Mode<br/>自进化模式|SprintCycle 自我优化与进化|🔴 极高|每次大版本必须人工审核、架构变更审批|框架自身能力提升|

## 三、Sprint 闭环工作流程

### 3.1 完整 Sprint 循环

**9个阶段完整闭环**：
1. **诊断 🔍** - 项目现状扫描、问题识别与优先级排序
2. **意图生成 🎯** - 基于诊断生成进化意图，人工确认调整
3. **Sprint 规划 📋** - 任务拆解、定义验收标准与成功指标
4. **PRD 生成 ⚡** - 结合知识沉淀生成结构化PRD
5. **编码执行 ⚡** - 选择单一编码引擎保证一致性，多轮生成调整
6. **验证与修复 ✅** - 自动化测试执行，智能修复，最多重试3次
7. **结果评估 📊** - 多维度评分，与规划目标对比分析
8. **知识沉淀 🧠** - 经验提炼、更新模式库、优化未来策略
9. **报告输出** - 生成完整迭代报告

## 四、技术架构设计

### 4.1 三层核心架构

**Layer 1: API 层（统一入口）**
- SprintCycle 核心类 6 大方法：plan()、run()、diagnose()、status()、rollback()、stop()
- 多入口支持：Web Dashboard、MCP Server、CLI 命令行

**Layer 2: 执行层（核心复用）**
- SprintExecutor 统一执行器 - 所有策略共用一套执行逻辑
- 基础组件：EventBus 事件总线、StateStore 状态存储、Checkpoint 检查点、FeedbackLoop 反馈闭环

**Layer 3: 能力层（策略可插拔）**
- EvolutionStrategy 进化策略工厂：SelfEvolutionStrategy（框架自身进化）、ProductEvolutionStrategy（用户产品进化）
- QualityGuard 质量守护：ArchitectureGuard + EvolutionFitness + TestEngine

### 4.2 三条进化主线与架构不变量

**📈 主线1：交付效率持续提升**
- 可进化：Agent优化、缓存策略、任务调度算法
- 不可动：9阶段流程顺序、统一API接口签名

**🎯 主线2：代码质量持续提升**
- 可进化：验证规则智能化、测试覆盖率自动提升
- 不可动：四级验证体系、Checkpoint回滚机制

**🏗️ 主线3：自身架构持续完善**
- 可进化：性能优化、Bug修复、新能力扩展
- 不可动：多引擎策略模式、三层依赖架构

**核心架构不变量清单（绝对不可修改）**：

|类别|不变量内容|保护级别|
|---|---|---|
|设计模式|策略模式（Strategy）用于多引擎|🔴 最高|
||三层架构分层依赖原则|🔴 最高|
|流程顺序|多阶段 Sprint 执行顺序不可颠倒，不可增减|🔴 最高|
|分层依赖|API层 → 执行层 → 能力层，不可反向依赖|🟠 高|
|API签名|SprintCycle 核心6大方法签名|🟠 高|
|数据结构|SprintResult、Checkpoint、DiagnosticReport|🟡 中|

## 五、部署与生态方案

### 5.1 一键跨平台部署

**我们已经实现的部署能力**：
- ✅ macOS（Intel / Apple Silicon）全自动
- ✅ Linux（Ubuntu / Debian / CentOS / Fedora / Arch）全自动
- ✅ curl | bash 远程一键部署
- ✅ 5分钟完成从 0 到可运行开发环境

**任意 Mac / Linux 机器上只需一行**：

```bash
curl -fsSL https://raw.githubusercontent.com/sprintcycle/sprintcycle/main/dev-setup.sh | bash
```

### 5.2 企业级自部署方案

**三层企业部署架构**：
1. **用户访问层**：Web UI Dashboard、OpenClaw 插件、CLI 命令行、IDE 插件
2. **服务层**：API 网关、任务调度、执行队列、结果存储
3. **数据与模型层**：PostgreSQL 元数据、Redis 缓存、LLM / 模型网关

**🔒 安全特性**：全链路数据加密、API Key 安全存储、访问审计、内网隔离

## 八、总结：SprintCycle vs Y Build 核心差异

|维度|Y Build|SprintCycle|
|---|---|---|
|开源状态|❌ 闭源 SaaS|✅ 完全开源|

|自进化|❌ 无|✅ 内置自进化闭环|
|企业审计|❌ 无|✅ 完整审计与追踪|
|目标用户|非技术创始人、 Indie Hacker|全用户覆盖：从创始人到企业开发团队|

**核心价值主张**：  
*"Y Build 让你快速开发，SprintCycle 让你在快速开发的同时，还能保证代码质量并完全掌控数据。"*
